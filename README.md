## **Key Architecture Decisions**

**1. Serverless-First Strategy**

- **Architecture Choice**: AWS Lambda over EC2/containers
- **Business Rationale**: Eliminate operational overhead, optimise TCO
- **Technical Justification**: Automatic scaling, pay-per-use model, reduced maintenance

**Enterprise Impact**: Designed for 70%+ TCO reduction through a serverless pay-per-use model

**2. Event-Driven Pattern**

- **Architecture Choice**: S3 event notifications over polling mechanisms
- **Business Rationale**: Real-time processing, reduced operational complexity
- **Technical Justification**: Loose coupling, built-in retry mechanisms, scalability
- **Performance Impact**: Sub-30-second processing from upload to storage

**3. Infrastructure as Code**

- **Architecture Choice**: Terraform for resource provisioning
- **Business Rationale**: Reproducibility, audit trail, team collaboration
- **Technical Justification**: Version control, environment consistency, compliance enforcement
- **Governance Impact**: Standardised deployment processes across environments

**4. Security by Design**

- **Architecture Choice**: Least privilege IAM roles and policies
- **Business Rationale**: Risk mitigation, compliance readiness, and audit compliance
- **Technical Justification**: Minimal attack surface, principle of least privilege
- **Enterprise Value**: Production-ready security posture
